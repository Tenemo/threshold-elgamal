<section class="landing-section not-content">
  <div class="landing-section__header">
    <h2>Follow the supported ceremony</h2>
    <p>
      The library is intentionally opinionated. The supported path is short and
      explicit from manifest publication to final tally.
    </p>
  </div>

  <div class="landing-steps-shell">
    <Steps>
      1. **Publish the manifest and accepted roster.**
         The public manifest surface is intentionally minimal: `rosterHash` and
         `optionList`.
      2. **Run honest-majority DKG.**
         The threshold is derived from the accepted roster as `ceil(n / 2)` and
         backed by verifiable checkpoints.
      3. **Accept fixed score ballots and close the board.**
         Ballots always use complete `1..10` score assignments and the organizer
         closes counting with a signed `ballot-close`.
      4. **Publish shares, tally, and verify the board.**
         The verifier checks the full ceremony transcript instead of isolated
         payload fragments.
    </Steps>
  </div>
</section>
