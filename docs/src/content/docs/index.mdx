    </p>

        <p>
          This library helps a group run a vote where ballots stay secret while
          voting is open, no single person can reveal the result alone, and the
          final public record can be checked by auditors or the voters themselves.
        </p>
        <p>
          You can think of it as the cryptographic engine behind a public vote:
          it creates the locked ballots, the signed messages that move between
          steps, and the proof material needed to trust the published tally.
        </p>
      </div>
    </Card>

    <Card title="Technical details">
      <div class="landing-card-copy">
        <p>
          <code>threshold-elgamal</code> is a browser-native TypeScript library
          for additive ElGamal over Ristretto255, signed protocol payloads,
          deterministic serialization, and honest-majority threshold
          decryption.
        </p>
        <p>
          The supported workflow covers minimal manifest publication, GJKR DKG
          with verifiable checkpoints, fixed <code>1..10</code> score ballots,
          ciphertext aggregation, decryption-share publication, and full
          public-board verification.
        </p>

<section class="landing-section not-content">
  <div class="landing-section__header">
    <h2>Follow the supported ceremony</h2>
    <p>
      The library is intentionally opinionated. The supported path is short and
      explicit from manifest publication to final tally.
    </p>
  </div>

  <div class="landing-steps-shell">
    <Steps>
      1. **Publish the manifest and accepted roster.**
         The public manifest surface is intentionally minimal: `rosterHash` and
         `optionList`.
      2. **Run honest-majority DKG.**
         The threshold is derived from the accepted roster as `ceil(n / 2)` and
         backed by verifiable checkpoints.
      3. **Accept fixed score ballots and close the board.**
         Ballots always use complete `1..10` score assignments and the organizer
         closes counting with a signed `ballot-close`.
      4. **Publish shares, tally, and verify the board.**
         The verifier checks the full ceremony transcript instead of isolated
         payload fragments.
    </Steps>
  </div>
</section>
